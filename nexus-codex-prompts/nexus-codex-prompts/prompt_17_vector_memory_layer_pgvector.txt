PROMPT 17 — NEXUS VECTOR MEMORY LAYER (Supabase pgvector)

You are a Senior Staff Engineer implementing the Nexus Vector Memory Layer using Supabase pgvector.

Purpose:
Create semantic memory so Nexus AI employees can search stored knowledge instead of repeatedly calling LLM APIs.

STRICT RULES:
- Add-only implementation
- Do not modify existing trading services
- No secrets committed
- Use Supabase pgvector extension
- Embeddings are generated by OpenClaw running on the external operator node (Mac Mini), then stored in Supabase

SUPABASE SETUP (SQL file):
/opt/nexus-services/ops/supabase/vector_memory.sql

1) Enable extension:
create extension if not exists vector;

2) Create table:
create table if not exists vector_memory (
  id uuid primary key default gen_random_uuid(),
  created_at timestamptz default now(),
  content text not null,
  content_type text not null,
  source_id uuid,
  source_table text,
  embedding vector(1536),
  meta jsonb default '{}'::jsonb
);

3) Create index:
create index if not exists vector_memory_embedding_idx
on vector_memory
using ivfflat (embedding vector_cosine_ops)
with (lists = 100);
